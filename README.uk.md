Slovo (українською — *«слово»*) — це крихітний перекладач із відчуттям рідної програми, який не заважає вам працювати.
Виділіть текст у **будь-якому** застосунку, натисніть глобальну гарячу клавішу — і з'явиться напівпрозоре спливне вікно з уже готовим перекладом, на основі [DeepL API](https://www.deepl.com/pro-api). Жодних вкладок браузера, жодних танців із копіюванням і вставлянням, жодного безладу. Slovo тихо живе у вашому menubar і важить майже нічого.

Створено на **Rust + Tauri v2** зі звичайним фронтендом на HTML/CSS/JS — без фреймворків, без кроку збірки, без npm.

## Можливості

- ⚡ **Глобальна гаряча клавіша** (за замовчуванням <kbd>⌘</kbd><kbd>⇧</kbd><kbd>T</kbd>, повністю **налаштовується** у застосунку) захоплює поточне виділення тексту з *будь-якого* застосунку та миттєво перекладає його.
- 🌐 **100+ мов** з автоматичним визначенням мови оригіналу, що завантажуються наживо з DeepL.
- ✍️ **Автопереклад під час набору** — із затримкою (debounce), без жодної кнопки.
- 🪟 **Рідне оформлення macOS** — справжній рядок заголовка зі «світлофором», напівпрозорий ефект vibrancy/розмиття, світла й темна теми, а також вікно, що підлаштовується під свій вміст.
- 📍 **Живе в menubar.** Закриття вікна ховає його з Dock; гаряча клавіша або значок у menubar миттєво повертають його назад.
- 🚀 **Необов'язковий запуск при вході в систему.**
- 🔒 **Приватність за задумом.** Ваш ключ DeepL зберігається локально (`~/Library/Application Support/slovo/config.json`); до DeepL потрапляють лише ваш текст і цей ключ — і нічого більше.

## Як це працює

Коли ви натискаєте гарячу клавішу, Slovo:

1. Імітує <kbd>⌘</kbd><kbd>C</kbd> в активному застосунку, щоб скопіювати поточне виділення.
2. Зчитує отриманий текст із буфера обміну (а потім відновлює попередній вміст буфера).
3. Надсилає текст до DeepL, який автоматично визначає мову оригіналу та перекладає його.
4. Показує напівпрозоре спливне вікно з результатом.

Якщо нічого не виділено, спливне вікно просто відкривається порожнім і готовим до введення.

> **Дозвіл Accessibility.** Оскільки крок 1 синтезує натискання клавіш в іншому
> застосунку, macOS вимагає, щоб Slovo було надано доступ до **Accessibility**
> (*System Settings → Privacy & Security → Accessibility*). Панель налаштувань у застосунку
> показує поточний стан дозволу та кнопку **Grant** для надання доступу в один клік. Без нього
> гаряча клавіша відкриватиме спливне вікно, але не зможе захопити ваше виділення.

## Встановлення

### Варіант 1 — Завантажити реліз (рекомендовано)

1. Візьміть найновіший `Slovo_x.y.z_aarch64.dmg` зі сторінки [**Releases**](https://github.com/Miozzik/slovo/releases).
2. Відкрийте `.dmg` і перетягніть **Slovo** до своєї теки **Applications**.
3. Під час першого запуску **клацніть правою кнопкою миші по застосунку → Open**, потім підтвердьте.

> ⚠️ **Чому правий клік → Open?** Застосунок *підписано ad-hoc, але не нотаризовано*, тож
> Gatekeeper першого разу блокує звичайний подвійний клік. Правий клік → Open — це
> стандартний одноразовий спосіб обійти це обмеження; надалі застосунок запускається як зазвичай.

### Варіант 2 — Зібрати з вихідного коду

Див. [Збірка з вихідного коду](#збірка-з-вихідного-коду) нижче.

## Збірка з вихідного коду

### Передумови

- **macOS** на **Apple Silicon** (типова ціль збірки — `aarch64`).
  *Intel Mac:* додайте ціль командою `rustup target add x86_64-apple-darwin` і збирайте з `--target x86_64-apple-darwin`.
- **Rust** через [rustup](https://rustup.rs).
- **Xcode Command Line Tools**: `xcode-select --install`.
- **Tauri CLI**: `cargo install tauri-cli --version "^2"`.

> **Node.js не потрібен.** Фронтенд — це статичні HTML/CSS/JS, які Tauri віддає напряму.

### Кроки

```bash
git clone https://github.com/Miozzik/slovo.git
cd slovo/src-tauri
cargo tauri build
```

Результати:

- Бандл застосунку: `src-tauri/target/release/bundle/macos/Slovo.app`
- Інсталятор: `src-tauri/target/release/bundle/dmg/Slovo_0.1.0_aarch64.dmg`

Для розробки:

```bash
cargo tauri dev
```

> **Примітка:** Деяка поведінка (значок у menubar/рядку стану та приховування/показ у Dock) працює
> коректно лише під час запуску зібраного `.app`. Використовуйте `cargo tauri build` і запускайте
> бандл, щоб отримати повноцінний досвід.

> ⚠️ **Не запускайте `cargo update`.** Закомічений `Cargo.lock` навмисно закріплює крейт
> `time` на версії **`0.3.47`**. Версія `0.3.48` не компілюється на поточному `rustc`
> через помилку конфлікту трейтів `E0119`. Файл блокування закомічено навмисно — залиште
> його як є, і збірка залишатиметься «зеленою».

## Використання

1. **Отримайте ключ DeepL API.** Зареєструйте безкоштовний ключ на [deepl.com/pro-api](https://www.deepl.com/pro-api) (ключі безкоштовного тарифу закінчуються на `:fx` і повністю підтримуються).
2. **Відкрийте Налаштування** (значок шестерні) і вставте свій ключ.
3. **Надайте доступ Accessibility** — Налаштування показують поточний стан і кнопку **Grant**. Це дозволяє гарячій клавіші захоплювати ваше виділення.
4. **Призначте гарячу клавішу** на свій смак (за замовчуванням <kbd>⌘</kbd><kbd>⇧</kbd><kbd>T</kbd>).
5. **Увімкніть запуск при вході**, якщо хочете, щоб Slovo було готове одразу після входу в систему.

Тепер виділіть текст будь-де, натисніть свою гарячу клавішу — і спостерігайте за перекладом. Або просто відкрийте спливне вікно й почніть набирати — воно перекладатиме на ходу.

## Конфігурація

- **Файл конфігурації:** `~/Library/Application Support/slovo/config.json`
  Зберігає ваш ключ API та обрану комбінацію гарячої клавіші. Створюється автоматично, коли ви зберігаєте налаштування.
- **Запасний варіант через середовище:** якщо у файлі конфігурації не збережено ключ, Slovo звертається до
  змінної середовища `DEEPL_API_KEY`.

Кінцева точка обирається автоматично за вашим ключем: ключі, що закінчуються на `:fx`, використовують
безкоштовний хост API DeepL, усі інші — хост Pro.
